Del código local a la nube: CI/CD, infraestructura y seguridad.

1. Infraestructura: Local vs Nube

En desarrollo local, usamos Docker Compose para MongoDB y la API corre con npm run dev (nodemon).

En producción (Render), la API se despliega como servicio web y MongoDB se conecta a MongoDB Atlas (base de datos externa).

¿Por qué separar MongoDB? En Render (plan free), los datos se pierden al reiniciar el servicio. Usar MongoDB Atlas garantiza persistencia.

Tecnologías

  • Render: Hosting de la API (Node.js) y Frontend (Static Site)
  • MongoDB Atlas: Base de datos en la nube
  • Docker: Entorno de desarrollo reproducible

2. Configuración de Render

render.yaml (Infraestructura como Código)

services:
  - type: web
    name: share-notes-api
    env: node
    region: oregon
    plan: free
    rootDir: share-notes-backend
    buildCommand: npm install
    startCommand: npm start
    envVars:
      - key: NODE_ENV
        value: production
      - key: PORT
        value: 4000
      - key: MONGO_URI
        sync: false
      - key: JWT_SECRET
        sync: false
      # ... más variables

  - type: static
    name: share-notes-frontend
    buildCommand: npm run build
    staticPublishPath: ./dist
    rootDir: share-notes-frontend

Servicio Web (API)

  • Node.js en puerto 4000
  • Build: npm install
  • Start: npm start

Servicio Static (Frontend)

  • React + Vite
  • Build: npm run build
  • Publica la carpeta dist/

3. La Bóveda de Seguridad

Las variables sensibles (JWT_SECRET, MONGO_URI, GOOGLE_CLIENT_SECRET) nunca deben estar escritas en el código.

Usamos sync: false en render.yaml: declaramos que la variable existe, pero el valor real se escribe manualmente en el panel de Render.

Zero-Footprint

El archivo render.yaml solo reserva el nombre de la variable.

sync: false significa: "Humano, tú pones el valor secreto directamente en Render".

Nunca subas el .env a GitHub. Usa .env.example como plantilla.

4. CORS y Cookies en Producción

El frontend y backend están en dominios distintos. La cookie de autenticación DEBE configurarse correctamente:

// app.js - Configuración de CORS
app.use(cors({
  origin: ["http://localhost:5173", process.env.CLIENT_URL],
  credentials: true,           // Permite enviar cookies
  methods: ["GET", "POST", "PUT", "DELETE", "PATCH"],
  allowedHeaders: ["Content-Type", "Authorization", "Cookie"],
  exposedHeaders: ["Set-Cookie"],
}));

// Cookie de autenticación
res.cookie("token", token, {
  httpOnly: true,
  secure: isProduction,        // true en producción (HTTPS)
  sameSite: isProduction ? "none" : "lax",
  maxAge: 7 * 24 * 60 * 60 * 1000,
});

Conceptos Clave

  • httpOnly: No accesible desde JavaScript
  • secure: Solo se envía por HTTPS
  • sameSite: "none": Permite cross-origin
  • withCredentials: Axios debe tenerlo en true
Error común: Si falta sameSite: 'none' o secure: true, el navegador bloquea la cookie en producción.
