Docker: Contenedores y Portabilidad

Virtualización de sistema operativo para entornos de desarrollo y producción consistentes.

1. El Salto a los Contenedores

Virtualización de OS

A diferencia de las máquinas virtuales, los contenedores comparten el Kernel del Sistema Operativo del host.

VMs: Hardware virtualizado + OS completo (Pesado).
Contenedores: Aislamiento de procesos (Ligero/Rápido).

Ventajas Clave

  • Portabilidad: "Funciona en mi máquina y en la tuya".
  • Consistencia: Entorno idéntico de Dev a Prod.
  • Eficiencia: Arranque en segundos, mínimo consumo de RAM.

2. Arquitectura del Proyecto

Componentes

Docker Engine: El motor que corre los contenedores.

Imágenes: Plantillas de solo lectura (la "clase").

Contenedores: Instancias vivas de una imagen (el "objeto").

En Share Notes: Tenemos 2 servicios: mongodb (base de datos) y la API de Node.js.

docker-compose.yml

version: "3.8"

services:
  mongodb:
    image: mongo:7
    container_name: share-notes-mongodb
    restart: unless-stopped
    ports:
      - "27017:27017"
    environment:
      MONGO_INITDB_DATABASE: share_notes
    volumes:
      - mongodb_data:/data/db
    command: ["mongod", "--quiet", "--logpath", "/dev/null"]

volumes:
  mongodb_data:
    driver: local

3. Dockerfile: La Receta de la API

share-notes-backend/Dockerfile

FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .

EXPOSE 4000

CMD ["node", "src/server.js"]

Instrucciones

  • FROM node:20-alpine - Imagen base ultraligera
  • WORKDIR /app - Carpeta de trabajo
  • COPY package*.json ./ - Copia dependencias primero para cachear

Optimizaciones

  • npm ci --only=production - Solo dependencias de producción
  • Alpine Linux - Imagen ~170MB vs 1GB de node full
  • EXPOSE 4000 - Puerto de la API

4. Comandos Esenciales (CLI)

Imágenes

  • docker build -t share-notes-api .
  • docker images (Listar)
  • docker pull mongo:7 (Descargar)

Contenedores

  • docker-compose up -d (Iniciar todo)
  • docker ps (Ver activos)
  • docker stop share-notes-mongodb

Depuración

  • docker-compose logs -f
  • docker exec -it share-notes-mongodb sh
  • docker-compose down -v (Limpiar todo)

5. Flujo de Trabajo con Docker

Desarrollo Local

  1. Clonar el repositorio
  2. Ejecutar docker-compose up -d (levanta MongoDB)
  3. Ejecutar npm run dev en la API (nodemon)
  4. Ejecutar npm run dev en el frontend (Vite)

Producción (Render)

  1. Push a GitHub (main)
  2. Render detecta el cambio y construye
  3. Render ejecuta npm install && npm start
  4. API live en puerto 4000
